**Architecture now working:**

2. Memory populated: `["1A", "1B", "1C", "1D"]` question IDs (strings, not ints — backend returns e.g. `"1A"`)
3. Auto-solve waits for identify via `isIdentifyComplete` / `pendingAutoSolve` flags
4. 4 parallel solvers launch → send to `/api/v1/study/auto-solve` (stage: solve)
5. Answers arrive incrementally, sorted by id (`"1A" < "1B"…`), displayed in overlay

## Key Architecture Decisions Made

| Decision | Rationale |
|----------|-----------|
| Question IDs are `String` not `Int` | Backend returns `"1A"`, `"1B"` etc. — changed throughout Swift (`SolveResult`, `SessionMemory`, OverlayManager) |
| `pendingAutoSolveAnswers` only mutated on `@MainActor` | Background-thread read of shared array caused "0 answers" race — all `solversInFlight` mutations and array appends now in `await MainActor.run {}` |
| Answers sent to JS as `JSON array`, not JS string literals | LaTeX backslashes broke single-quoted JS string literals — now `JSONSerialization.data(withJSONObject:)` + `setAllAutosolveAnswers(jsonArr)` |
| Stop Solving doesn't cancel in-flight tasks | Cancelling mid-flight dropped answers — only `endSession()` cancels tasks |
| `isIdentifyComplete` + `pendingAutoSolve` flags | Auto-solve was running before identify completed, saw 0 questions |
| Incremental display: sort + render after each answer | Answers show as they arrive, always in id order |
